# Mysql Database Application

参考书：《数据库系统原理与Mysql应用教程》

# 第一章：数据库系统概述

**数据库组成**

数据库管理系统**（DBMS）**，用户，数据库



**数据库管理员DBA职责**

数据定义	DDL 	data definition language

数据操作	DML	data manipulation language



**应用系统DBAS**

在DBMS支持下的具体应用（前端）



**数据库的三级模式**

模式（逻辑模式）：就是整个全局的数据库，包含数据与逻辑

外模式（自模式，用户模式）：对外，面对一个具体的对象

内模式（存储模式）：对内，用于具体落实数据的存储



# 第二章：信息与数据模型

**数据模型**

概念层：是一个抽象的概念，常用E-R模型

逻辑层：**关系模型数据库**（主要），网状模型，层状模型

物理层：存储数据库的实际



**数据结构**

数据结构是定义数据的实际结构，非常重要



**数据操作**

增删改查



**完整性约束条件**

数据的要求，符合条件的数据才可以输入，保证数据的良好性



## **E-R模型**

实体【长方形】：老师，学生

属性【椭圆形】：name，age

联系【连线，一对一联系，一对多联系】：实体内部的联系，和实体之间的联系（主要）



**关系模型**

就是一张二维表

**代理键：**实体的唯一标示整数编码，递增，效率高

**关系模式R**（U，D，Dom，F）；关系名（属性，来自域，映像集合，依赖关系）



## 关系操作

数据查询：检索，统计，排序，分组，查询

数据维护：增删改

数据控制：约束性，完整性逻辑



### 关系代数8种操作

## SQL：Structure Query Language

关系代数：对运算查询进行表示

关系演算：对查询到的元祖进行表示
